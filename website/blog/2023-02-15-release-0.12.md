## Release Details


### Podman v4.4.1 [#1456](https://github.com/containers/podman-desktop/pull/1456)

Upgrade podman installers to v4.4.1


### allow to add port mappings when image has no exported port [#1265](https://github.com/containers/podman-desktop/pull/1265)

This PR allows to add port mappings when image has no exported port
Allow to add port mappings when starting an image that has no exported ports.

If there is no exported port, the user is free to add multiple local:remote port mappings

### Installation on Windows Home Edition [#1268](https://github.com/containers/podman-desktop/pull/1268)

It use Virtual Machine Platform detection, instead of hyper-v.
As Virtual Machine Platform is subset of Hyper-V, and required to use WSL2, this PR changes detection, which allow installation if podman on windows home edition.

![install-windows-home-edition-1](https://user-images.githubusercontent.com/929743/214841499-c188021d-b1ca-4ef8-80ff-90a4faa387cc.png)

### Minize on login [#1374](https://github.com/containers/podman-desktop/pull/1374)

On Windows & macOS, there is now an option to minimize on login.
Electron does this by reading the argsv value and seeing if --minimized
has been passed in through the macOS / Windows startup scripts.

#### fix: Update Pod status icon [#1326](https://github.com/containers/podman-desktop/pull/1326) and [#1377](https://github.com/containers/podman-desktop/pull/1377)

Currently, the status icons on the Images/Volumes/Pods list (rectangular icons using Font Awesome) don't match the Containers list (square, using a variety of svgs). After reviewing the designs and talking to @mairin we came up with the following guidelines:

All icons should be square, and slightly larger than the existing Containers icons.
All icons should use the associated pod/volume/etc SVGs (which means removing the cool but custom ones in the Container list :( ) to keep the mental association.
Stopped/created/unused icons should be grey outlines; started/used should be filled green and use solid icons.
This PR currently has only the changes to the Pod list - which would introduce a third list style without replacing either of the existing two. I have the rest of the changes ready, but I wanted to get confirmation of both the design and technical direction before committing them. If we agree on the direction, I can either expand this PR or submit a second one immediately.

This change rolls out the consistent status icon change started by PR #1326 to the other places that had status icons: container, image, and volume list, and the container details page.

Minor tweaks to layout to make the status columns identical in all 4 lists.

Old, now unused status icons are removed.

![pods-status-icon](https://user-images.githubusercontent.com/19958075/216671859-bdd8dca4-56b7-40a8-961a-dcb6e01be61e.png)
